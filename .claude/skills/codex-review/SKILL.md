---

codex CLI を使って現在のリポジトリの変更をコードレビューしてください。引数: 「$ARGUMENTS」

手順:

1. レビュー対象を判定する:
   - `git status --short` で未コミットの変更があれば `--uncommitted` を使う
   - 変更がコミット済みなら `--base <ベースブランチ>`(引数で指定があればそれ、なければ `main`)を使う

2. codex にレビューを委任する(数分かかる場合はバックグラウンド実行し、完了通知を待つ):

   ```bash
   codex exec review --uncommitted \
     -o "$SCRATCHPAD/codex-review.txt" \
     "追加の観点があればここに記載"
   ```

3. レビュー結果を読み、指摘を重要度順に整理してユーザーに報告する:
   - 各指摘に対象ファイル・行と、妥当かどうかの自分の判断を添える
   - 誤検知と思われる指摘はその理由とともに区別して示す

4. 修正はユーザーの指示があるまで行わない。修正を求められたら `/delegate` で codex に委任する。
