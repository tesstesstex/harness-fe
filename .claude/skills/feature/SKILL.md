---

「$ARGUMENTS」という機能を、以下のワークフローで開発してください。

1. **要件定義**: 機能を分析し、以下を簡潔にまとめてユーザーに提示する:
   - ユーザーストーリーと受け入れ条件
   - 影響範囲(ページ / API / コンポーネント)
   - テスト方針(単体 / E2E)
   - 曖昧な点があれば AskUserQuestion で確認する

2. **タスク分解**: 実装を自己完結した単位(1〜3個程度)に分割し、TaskCreate で登録する。

3. **実装委任**: 各タスクを `/delegate` スキルの手順に従って codex に委任する。
   指示文には要件・受け入れ条件・対象ファイル・「テストも書くこと」「npm run check が通ること」を含める。

4. **検証**: codex 完了ごとに `git diff --stat` と `npm run check` で検証する。
   UI に関わる変更なら `npm run test:e2e` も実行する。失敗したら失敗内容を添えて codex に再委任する。

5. **レビュー**: 全タスク完了後、`/codex-review` の手順で codex にレビューさせ、重要な指摘があれば修正を委任する。

6. **報告**: 実装内容・テスト結果・レビュー結果をまとめて報告する。コミットはユーザーの指示があるまで行わない。
